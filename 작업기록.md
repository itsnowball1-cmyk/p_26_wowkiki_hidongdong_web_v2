# 작업 기록 (2026-05)

하이동동 web — Cloudflare Workers 임시 스캐폴드에서 Node/Docker(NAS dev) 기반으로 전환하고,
운영 화면과의 차이를 맞춰온 작업 요약.

## 1. 아키텍처 / 배포 환경
- 백엔드 런타임: **Cloudflare Workers → Node http 서버** (`code/server/`)
  - `code/server/api.ts` : 워커 핸들러를 `handleRequest(request, env)` 로 export, CF 전용(`env.ASSETS`, `ExportedHandler`, `Fetcher`) 제거
  - `code/server/index.ts` : `node:http` ↔ Web `Request`/`Response` 브리지
  - wrangler/`@cloudflare/workers-types` 제거, `tsx` 도입. 스크립트: `dev`(vite) / `api`(node) / `build` / `start`
- **dev 배포 = Synology NAS (docker)**, prod = AWS (별개)
  - 경로 `/volume1/docker/hidongdong_web/`, 계정 `duckbest`
  - 포트: web `3011`(nginx, SPA+`/api` 프록시) / api `4011`(node) / db 내부 3306
  - URL: https://dev-hidongdong.intellicode.kr (DSM 리버스 프록시 → 3011)
  - 배포 자산: `deploy/nas/` (Dockerfile.api, Dockerfile.web, nginx.conf, docker-compose.yml)
- 로컬 dev: `docker-compose.yml`(MariaDB 10.3) + `npm run api`(4011) + `npm run dev`(3011)

## 2. DB
- 엔진: **MariaDB 10.3** (운영 원본과 동일, 컨테이너 고정)
- 현재 구성: **스키마 = dump-…271833 (승인 워크플로우 정식 스키마) / 데이터 = dump-…271543**
  - `db/schema-delta.sql` : 271543(구 스키마) 적재 후 271833 추가분(tb_member `approval_status`/`license_file_nm` + 신규 5테이블 `tb_institution`,`tb_approval_history`,`tb_license_file`,`tb_sms_template`,`tb_sms_verification`)을 얹음
  - `db/init.sh` : dump 첫 줄(MariaDB 11.x sandbox-mode) 제거 후 import
- 운영 dump(`backup/`, `dump-*.sql`)는 PII 포함 → **git 제외**. 팀원은 사내 채널로 받아 배치.
- NAS DB 는 로컬과 동일하게 mysqldump 로 동기화.

## 3. 화면/기능 수정 (운영 화면 기준으로 맞춤)
- **진단일/검사일 → 일시 표기**: `진단일`→`진단일시`, 검사 결과 상세 포함 `YYYY.MM.DD HH:MM`
- **치료일 → 치료일시**: 동일 시각 표기(+`parseTreatedAt` 보정)
- **진단 목록 사용시간 컬럼** 추가
- **오류 음소 및 위치**:
  - analysislog `e_list` 필드명 두 형식(`aim_joum/e_ctgr` ↔ `a_jm/ctgr`) 모두 인식하도록 정규화 — 축약형 553건 표시 누락 수정
  - 음소 등장 횟수 표기 `/ㄱ/ (3)` (레퍼런스 동일, cnt>1 일 때)
- **진단 이력 카드(자음정확도/단어 내 위치별/모음정확도)**: 더미값 제거 → 최신 진단 statistics 실제값(UTAP_PCC / TOTAL_PCC / PVC), 값 없으면 `--`
- **진단/치료 녹음 듣기 음성파일 링크**: 레퍼런스 규칙으로 통일
  - `tb_data_list`↔`tb_data_file` 조인, 폴더(`idx_{id}`) 내 전체 파일 나열
  - url = `${FILES_BASE_URL}/dataCenter${file_nm}` (운영: `https://api.wowkiki.kr`)

## 4. 알려진 이슈 / 메모
- 치료사 mtype: 코드는 `healler` 조회, 일부 데이터는 `teacher` — 데이터/코드 정합 별도 점검 필요
- `diag_days` 컬럼은 코드가 try/catch 로 부재 허용(정식 스키마에 없음)
- PVC(모음정확도)는 데이터에 없는 진단이 많아 카드에 `--` 로 표기되는 경우 다수 — 정상
- api/vite 를 cmd 창으로 실행 → 창 닫으면 종료. 상시 실행은 pm2 등 별도 구성 권장
